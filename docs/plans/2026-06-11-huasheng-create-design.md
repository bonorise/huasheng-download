# 华声项目自动创建设计

## 目标

新增一个独立脚本。用户传入 TXT 文件路径后，脚本复用项目现有的华声登录态，在华声首页创建项目，依次选择“A - 素材剪辑成片”并提交两次“确认”。任务完成后在终端输出新项目 URL，同时保留浏览器现场，直到用户按 `Ctrl+C` 结束进程。

新功能与现有素材下载、MG 动画下载模块相互独立，不修改已有下载流程。

## 命令入口

新增独立命令：

```bash
npm run create -- /绝对路径/文案.txt
```

支持以下可选参数：

```text
--profile <目录>
--headless
--slow-mo <毫秒>
```

默认使用可见浏览器，并复用现有 `.browser-profile` 持久化登录态。

## 模块结构

新增 `src/huasheng-create.js`，负责：

- 解析创建命令参数。
- 读取并验证 TXT 文件。
- 驱动华声网页完成项目创建。
- 输出新项目 URL。
- 在成功或失败后保留浏览器和进程。

复用 `src/shared.js` 中已有能力：

- `launchBrowser()`：启动使用持久化登录态的 Playwright 浏览器。
- `isProbablyLoggedOut()`：检测登录状态。
- `pauseForEnter()`：等待用户手动完成登录。

不修改 `src/index.js`、`src/huasheng-download.js` 或 `src/mg-download.js` 的业务逻辑。

## 输入处理

脚本启动浏览器前完成以下检查：

1. 必须提供 TXT 文件路径。
2. 文件必须存在且可读取。
3. 读取为 UTF-8 文本。
4. 使用 `trim()` 去除首尾空白，保留正文内部换行。
5. 去除首尾空白后内容不能为空。

任一检查失败时直接报错退出，不启动浏览器。

## 自动化流程

1. 打开 `https://www.huasheng.cn/`。
2. 等待首页主文本输入框出现。
3. 若检测到未登录：
   - 可见浏览器下提示用户手动登录。
   - 用户在终端按回车后重新打开首页并继续。
   - 无头模式下直接进入失败处理。
4. 将处理后的 TXT 内容填入首页主文本输入框。
5. 点击文本为“创建”的按钮。
6. 最长等待 60 秒，检测 `A - 素材剪辑成片` 按钮；出现后立即点击。
7. 最长等待 60 秒，检测 `placeholder="输入你的任何想法"` 的对话输入框。
8. 在输入框中填入“确认”，按 Enter 提交，并等待输入框清空。
9. 最长等待 60 秒，等待页面进入符合以下格式的视频项目 URL：

   ```text
   https://www.huasheng.cn/video/<数字项目ID>?clip=-1
   ```

10. 记录该项目 URL。
11. 在视频项目页面重新定位对话输入框。
12. 填入“确认”，按 Enter 提交，并等待输入框清空。
13. 在终端输出项目 URL 和完成提示。
14. 保持浏览器与脚本进程运行，等待用户按 `Ctrl+C` 结束。

所有等待均采用状态检测，目标出现后立即继续，不固定等待完整 30 秒。单次最长等待时间为 60 秒。

## 元素定位策略

优先使用语义和稳定属性定位，避免依赖构建后可能变化的 CSS 哈希类名。

### 首页输入框

优先选择首页可见且面积最大的 `textarea.ant-input`，后备为可见的 `textarea`。这样既利用 Ant Design 的稳定基础类，也避免依赖 `css-1pueqiu`。

### 创建按钮

优先使用按钮角色和精确文本“创建”定位，后备为可见按钮中的精确文本匹配，不依赖 `css-yacn5n`。

### 成片方式按钮

使用按钮角色和精确文本 `A - 素材剪辑成片` 定位。此步骤不兼容其他文案。

### 对话输入框

使用：

```css
textarea[placeholder="输入你的任何想法"]
```

每次填写和提交前重新获取当前可见输入框，以适应页面重渲染。

### 视频项目 URL

成功 URL 必须同时满足：

- 协议为 HTTPS。
- 域名为 `www.huasheng.cn`。
- 路径符合 `/video/<数字项目ID>`。
- 查询参数 `clip` 等于 `-1`。

## 提交成功判定

两次“确认”均使用以下标准：

1. 填入“确认”。
2. 按 Enter。
3. 最长等待 60 秒，确认输入框值清空。

若原输入框因页面重渲染被替换，只要当前匹配的新输入框值为空，也视为提交成功。

第二次提交完成且输入框清空后，整个自动化任务即成功，不再等待额外页面元素。

## 重试策略

以下关键步骤各允许重试一次：

- 打开首页。
- 填写文案。
- 点击创建。
- 点击 `A - 素材剪辑成片`。
- 第一次提交“确认”。
- 等待视频项目页。
- 第二次提交“确认”。

首次失败后重新定位元素，并完整执行当前步骤。第二次仍失败则进入最终失败处理。

## 错误处理

最终失败时输出：

- 失败步骤。
- 当前页面 URL。
- 简明错误原因。
- 浏览器已保留、需要人工检查的提示。

随后保持浏览器和脚本进程运行，不自动关闭。用户使用 `Ctrl+C` 结束。

成功时行为一致：输出新项目 URL 和完成提示后保留浏览器，等待用户使用 `Ctrl+C` 结束。

无头模式下无法人工完成登录或检查页面。若检测到未登录则报错，并按统一失败规则保持进程运行。
