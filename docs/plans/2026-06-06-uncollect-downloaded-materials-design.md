# 收藏素材下载后自动取消收藏设计

## 背景

华声的“收藏”素材是跨项目共享的全局列表。当前下载器会读取收藏列表中的素材并下载，但不会修改收藏状态，导致下次运行时重复下载同一批素材。

本次改动要求：收藏素材全部下载完成后，再启动独立的取消收藏阶段，点击本次成功下载素材封面卡片上的星标按钮。下载失败或使用 `--dry-run` 时必须保留收藏。

## 目标

- 仅在“收藏”模式中启用自动取消收藏。
- 收藏下载和取消收藏是两个独立模块，按固定顺序先后执行。
- 只有本轮全部下载完成并写入清单后，才启动取消收藏模块。
- 取消收藏模块只处理文件成功写入输出目录的素材。
- 精确定位已下载素材所属的卡片，避免误取消其他素材。
- 单个素材取消失败时继续处理后续素材，并保留可追踪的失败记录。
- 不改变“推荐”模式现有行为。

## 非目标

- 不把取消收藏逻辑嵌入单个文件的下载流程。
- 不为历史已下载文件自动清理收藏。
- 不在 `--dry-run` 模式中修改网页状态。
- 不增加控制自动取消收藏的命令行参数。

## 模块边界

### 收藏下载模块

保留当前“先提取全部素材 URL，再统一下载”的流程。它负责素材提取、文件下载和下载结果落盘，不调用星标点击逻辑。

提取收藏素材时，为每条素材记录保存可用于后续重新定位原卡片的封面特征。下载模块处理完所有素材后，先将最终下载状态写入 `manifest.json`。

### 取消收藏模块

取消收藏模块在收藏下载模块完整结束后启动。它读取本次内存中的 manifest 结果，筛选 `status === "downloaded"` 的收藏素材形成清理队列，然后逐个重新定位卡片并点击星标。

两个模块通过 manifest 条目中的下载状态和卡片签名传递数据，不通过下载回调相互调用。取消收藏结果会追加回同一条 manifest 记录，但不会修改原有下载状态。

星标使用前缀选择器匹配：

```css
[class*="ClipChoiceItem_collectIconWrap__"]
```

不依赖完整的 CSS Modules 哈希类名，避免构建后缀变化导致失效。

## 数据流

1. 打开项目并进入“收藏”tab。
2. 扫描素材卡片，点击封面获取视频 URL。
3. 提取时记录视频 URL、封面 URL、候选元素特征和卡片定位信息。
4. 完成收藏列表提取后，按现有流程逐个下载文件。
5. 所有文件处理结束后，将完整下载结果写入 `manifest.json`。
6. 如果不是 `--dry-run`，取消收藏模块筛选本次所有 `downloaded` 条目。
7. 使用每条记录保存的特征在收藏列表中重新定位素材卡片。
8. 在该卡片内部点击收藏星标。
9. 等待并确认该卡片从收藏列表消失，或星标状态发生预期变化。
10. 将取消收藏状态追加写入 `manifest.json`。

如果本轮没有成功下载项，或使用 `--dry-run`，则整个取消收藏阶段直接跳过。

## 卡片定位

提取候选封面时，同时向上查找包含收藏星标的最近祖先元素，将其视为素材卡片。记录以下信息：

- 封面资源地址。
- 候选元素标签和必要属性。
- 用于本次页面会话定位的卡片标识。

取消收藏时优先按封面资源地址重新查找元素，再向上定位包含星标的素材卡片。临时 DOM 标识仅作为同一页面未重渲染时的快速路径，不能作为唯一定位依据。

所有星标点击都必须限定在已匹配的卡片内部，不能直接点击页面中的第一个星标。

## 状态与错误处理

取消收藏阶段开始前，本轮收藏素材的 `uncollectStatus` 均为 `skipped`。取消收藏模块只更新成功下载条目：

- `uncollectStatus: "uncollected"`：下载成功且确认取消成功。
- `uncollectStatus: "failed"`：下载成功，但未能定位、点击或确认星标状态。
- `uncollectStatus: "skipped"`：下载失败或处于 `--dry-run`。

取消收藏失败时：

- 已下载文件保持不变。
- 下载记录仍为 `downloaded`。
- 将失败原因写入该条 manifest 记录和 `failures.json`。
- 继续处理后续素材。

下载模块即使部分失败也必须正常结束；取消收藏模块随后只处理成功队列，不受失败项影响。

如果进入取消收藏阶段时页面状态丢失，可重新打开项目、展开素材面板并选择“收藏”tab，再尝试一次定位。重试后仍失败则记录失败，不进行不确定的点击。

## 列表变化处理

取消收藏后卡片会从全局收藏列表中移除，列表可能重新排列。定位逻辑不得依赖最初的序号或坐标，而应每次基于保存的封面特征重新查询当前 DOM。

取消收藏队列在下载阶段结束后一次性生成，列表重排不会改变队列。后续素材取消收藏时重新定位即可。

## 测试

### 单元测试

- 验证 CSS Modules 类名前缀识别逻辑。
- 验证素材定位特征的规范化和匹配逻辑。
- 验证下载、`--dry-run` 和失败状态对应的取消收藏决策。
- 验证下载模块结束前不会调用取消收藏模块。
- 验证取消收藏队列只包含本轮 `downloaded` 条目。

### 浏览器验证

- 收藏两个以上素材并运行真实下载。
- 确认全部下载结束后，成功下载的素材才开始从“收藏”tab 消失。
- 制造一个下载失败，确认对应素材仍保留收藏。
- 使用 `--dry-run`，确认所有素材仍保留收藏。
- 使用 `--tab 推荐`，确认不会点击任何收藏星标。

## 验收标准

- 所有收藏素材完成下载处理后，才开始点击星标。
- 取消收藏模块只处理本轮成功下载的素材。
- 下载失败和 `--dry-run` 不会取消收藏。
- 取消失败不会影响已经完成的下载结果，也不会中止后续取消任务。
- 取消失败不会改变对应素材的 `downloaded` 状态。
- 不会因列表重排而取消错误素材。
- manifest 和失败记录可以区分下载结果与取消收藏结果。
- 推荐模式行为保持不变。
