# 收藏素材下载后自动取消收藏设计

## 背景

华声的“收藏”素材是跨项目共享的全局列表。当前下载器会读取收藏列表中的素材并下载，但不会修改收藏状态，导致下次运行时重复下载同一批素材。

本次改动要求：收藏素材文件下载成功后，点击该素材封面卡片上的星标按钮取消收藏。下载失败或使用 `--dry-run` 时必须保留收藏。

## 目标

- 仅在“收藏”模式中启用自动取消收藏。
- 只有文件成功写入输出目录后，才取消对应素材的收藏状态。
- 精确定位已下载素材所属的卡片，避免误取消其他素材。
- 单个素材取消失败时继续处理后续素材，并保留可追踪的失败记录。
- 不改变“推荐”模式现有行为。

## 非目标

- 不重构为“提取一个、下载一个”的实时处理流程。
- 不为历史已下载文件自动清理收藏。
- 不在 `--dry-run` 模式中修改网页状态。
- 不增加控制自动取消收藏的命令行参数。

## 方案

保留当前“先提取全部素材 URL，再统一下载”的流程。提取收藏素材时，为每条素材记录保存可用于重新定位原卡片的封面特征；下载成功后回到收藏列表，通过这些特征定位对应卡片，再点击卡片内部的收藏星标。

星标使用前缀选择器匹配：

```css
[class*="ClipChoiceItem_collectIconWrap__"]
```

不依赖完整的 CSS Modules 哈希类名，避免构建后缀变化导致失效。

## 数据流

1. 打开项目并进入“收藏”tab。
2. 扫描素材卡片，点击封面获取视频 URL。
3. 提取时记录视频 URL、封面 URL、候选元素特征和卡片定位信息。
4. 完成收藏列表提取后，按现有流程逐个下载文件。
5. 文件下载成功后，使用保存的特征在收藏列表中重新定位素材卡片。
6. 在该卡片内部点击收藏星标。
7. 等待并确认该卡片从收藏列表消失，或星标状态发生预期变化。
8. 将下载状态和取消收藏状态写入 `manifest.json`。

下载失败或 `--dry-run` 时跳过第 5 至第 7 步。

## 卡片定位

提取候选封面时，同时向上查找包含收藏星标的最近祖先元素，将其视为素材卡片。记录以下信息：

- 封面资源地址。
- 候选元素标签和必要属性。
- 用于本次页面会话定位的卡片标识。

取消收藏时优先按封面资源地址重新查找元素，再向上定位包含星标的素材卡片。临时 DOM 标识仅作为同一页面未重渲染时的快速路径，不能作为唯一定位依据。

所有星标点击都必须限定在已匹配的卡片内部，不能直接点击页面中的第一个星标。

## 状态与错误处理

每条收藏素材在清单中增加取消收藏结果：

- `uncollectStatus: "uncollected"`：下载成功且确认取消成功。
- `uncollectStatus: "failed"`：下载成功，但未能定位、点击或确认星标状态。
- `uncollectStatus: "skipped"`：下载失败或处于 `--dry-run`。

取消收藏失败时：

- 已下载文件保持不变。
- 下载记录仍为 `downloaded`。
- 将失败原因写入该条 manifest 记录和 `failures.json`。
- 继续处理后续素材。

如果下载成功后页面状态丢失，可重新打开项目、展开素材面板并选择“收藏”tab，再尝试一次定位。重试后仍失败则记录失败，不进行不确定的点击。

## 列表变化处理

取消收藏后卡片会从全局收藏列表中移除，列表可能重新排列。定位逻辑不得依赖最初的序号或坐标，而应每次基于保存的封面特征重新查询当前 DOM。

由于提取阶段已经收集全部待下载素材，列表重排不会改变下载队列。后续素材取消收藏时重新定位即可。

## 测试

### 单元测试

- 验证 CSS Modules 类名前缀识别逻辑。
- 验证素材定位特征的规范化和匹配逻辑。
- 验证下载、`--dry-run` 和失败状态对应的取消收藏决策。

### 浏览器验证

- 收藏两个以上素材并运行真实下载。
- 确认每个成功下载的素材随后从“收藏”tab 消失。
- 制造一个下载失败，确认对应素材仍保留收藏。
- 使用 `--dry-run`，确认所有素材仍保留收藏。
- 使用 `--tab 推荐`，确认不会点击任何收藏星标。

## 验收标准

- 收藏素材成功下载后，对应星标被点击并确认取消收藏。
- 下载失败和 `--dry-run` 不会取消收藏。
- 取消失败不会中止其他素材下载。
- 不会因列表重排而取消错误素材。
- manifest 和失败记录可以区分下载结果与取消收藏结果。
- 推荐模式行为保持不变。
