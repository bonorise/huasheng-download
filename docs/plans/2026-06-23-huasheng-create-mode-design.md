# 华声创建流程 A/B 方案选择改造设计

## 背景

华声网站创建视频项目的流程发生变化。用户提交文案后，网站会先在对话中要求选择制作方案：

- A - 素材剪辑成片
- B - 素材混合 MG 动画

旧脚本固定点击 `A - 素材剪辑成片` 按钮。现在按钮不一定稳定显示，直接在对话输入框里输入 `A` 或 `B` 更可靠。选择方案后，网站还会要求确认继续；既可以点击“确认并继续”，也可以直接在输入框输入“确认”。

此外，进入网站时可能不定期弹出站内消息弹窗。该弹窗不可控、暂时无法稳定复现，脚本需要在关键步骤前后做非侵入式清理。

## 目标

让 `src/huasheng-create.js` 支持显式选择 A/B 方案，并在没有指定时默认选择 A；选择和确认都优先通过对话输入框提交，减少对按钮显示状态的依赖。

## 非目标

- 不解析 TXT 文案里的控制行，避免把控制信息和创作正文混在一起。
- 不为未知消息弹窗做强绑定选择器，因为目前没有稳定复现样本。
- 不改变下载素材、MG 下载、分镜扫描等流程。

## 方案

### 参数与默认值

新增命令行参数：

```bash
npm run create -- /path/to/script.txt --mode B
```

`--mode` 只接受 `A` 或 `B`，大小写归一为大写。不提供时默认 `A`。Agent 在自然语言任务里识别到“用 B 方案”时传 `--mode B`；没有明确指定时不传参数，脚本默认 A。

### 交互流程

保留原有步骤：

1. 打开首页并复用 `.browser-profile` 登录态。
2. 填写 TXT 文案。
3. 点击“创建”按钮。

修改后续步骤：

1. 等待对话输入框可见。
2. 输入 `args.mode`，按 Enter。
3. 输入 `确认`，按 Enter。
4. 等待跳转到 `https://www.huasheng.cn/video/<id>?clip=-1`。
5. 如页面仍要求确认，继续输入一次 `确认`。

其中第 4、5 步保留现有“等待项目 URL + 第二次确认”结构，降低站点异步流程变化带来的风险。

### 弹窗处理

新增一个轻量 `dismissTransientOverlays(page)`：

- 尝试点击常见关闭控件，例如 `aria-label` 为关闭的按钮、文本为 `×` / `关闭` / `我知道了` 的可见按钮。
- 尝试按一次 `Escape`。
- 所有失败都吞掉，只作为重试前后的清理动作，不改变主流程判断。

该函数在 `runStepWithRetry` 的失败重试间隙，以及提交 A/B 或确认前调用。这样即使未知弹窗偶发遮挡输入框，第二次尝试也有机会恢复。

## 测试

自动化测试只覆盖不访问华声网站的纯逻辑：

- `parseCreateArgs` 默认 `mode` 为 `A`。
- `parseCreateArgs` 接受 `--mode B`。
- `parseCreateArgs` 将小写 `b` 归一为 `B`。
- 非 `A` / `B` 的 mode 报错。

浏览器端流程使用人工验收：

```bash
npm run create -- /path/to/script.txt
npm run create -- /path/to/script.txt --mode B
```

预期：

- 未指定时，网站收到 `A`。
- 指定 `--mode B` 时，网站收到 `B`。
- 方案选择后输入 `确认` 能继续到项目页。
